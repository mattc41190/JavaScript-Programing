There are three ways to make objects in JavaScript

  2. Using the new keyword

Object Literals

An object literal is the easiest way (though no way is particularly difficult) to create a JavaScript. With an object literal you both create and assign an object at once.

The example below display how to create an object in JavaScript using the object literal approach

age:50,
eyeColor:"blue"
};

The "This" Keyword

The "this" keyword in Javascript refers to the objects that owns the code currently running.The value of this, when used in a function, is the object that "owns" the function. The value of this, when used in an object, is the object itself. The this keyword in an object constructor does not have a value. It is only a substitute for the new object.


The "this" keyword is reserved for JavaScript and it cannot be reassigned to hold another value. Earlier a car object was created using the constructor object creation method. Using that same code with the addition of an object method the value of the "this" keyword will be explained.

function car(make, model, year, transmission) {
this.make = make;
this.model = model;
this.year = year;
this.transmission = transmission;

this.modelYear = function()
{
console.log(this.model +", "+this.year);
}
}

var firstCar = new car("Toyota", "Truck X50", 1989, "Auto");
var currentCar = new car("Ford", "Focus", 2008, "Auto");

When the car(params) function is called it is passed a number of parameters if any parameter is omitted it will be assigned a value of "undefined". So when the "firstCar" object is created that/"this" instance of that object is assigned unique values. So, when firstCar.make is called it returns "Toyota" and when currentCar is called it returns "Ford".

Object References

In JavaScript objects point to references not to values.

If animal is an object, the following statement will not create a copy of animal:


function creature(name, size,color,avgLife) {
this.name = name;
this.size = size;
this.color = color;
this.avgLife = avgLife;
}
var animal = new creature("cat","small","varies",15);
var beast = animal;

beast.avgLife = 15;
console.log(animal.avgLife); //This will return 15

Instead of beast being a copy of animal, beast actually points to the same place in memory that the animal object does. So if a property of beast is changed it will also be changed for animal.

Or it may be accessed using bracket notation as in person["age"] or person["name"]. The latter notation also lends itself to using variable with the correct expression as in var x = "age"; person[x];


The code below will demonstrate the ways to access object properties on a new object
function silverware(type,material,weight)

this.weight = weight

Looping Properties

In a JavaScript object the properties be looped through via a "For in... Loop". This takes all properties in the order they are declared and performs some operation against each property. This is useful in situations where each property to be presented needs to be designated to certain area based on one or more properties


The code below will demonstrate the result of looping through two silverware objects
function silverware(type,material,weight)
{
this.type = type
this.material = material
this.weight = weight
}

var mattSilverware = new silverware("fork","steel", "2oz");
var arielSilverware = new silverware("spoon","plastic", "1.5oz");
var result = [];

var silverwareArray = [mattSilverware, arielSilverware]

for(i in silverwareArray)
{
var currentObject = silverwareArray[i]
for(property in currentObject) // property === current object's properties and currentObject === the object itself.
{ result.push(currentObject[property]); }

Default Methods

In JavaScript there is a large number of built in methods that can be applied to properties. The methods are often specific to a certain data type. An example of a built in method is the toUpperCase() method. That method takes a string data type and converts it to an upper case version of itself. Another built in method is the toFixed(x) method. The toFixed(x) method takes a floating point number and displays that number rounded to the number passed into the method arguments section.


The code below will demonstrate the use of JavaScript's built-in method:
var building = {
material:"brick",
height: 15,
length: 40,
width: 25,
unitCost: 4.5,
cost: function()
{
return this.unitCost * this.height * this.length * this.width;
}
};

document.getElementById('toFixedMethodExample').innerHTML = building.unitCost.toFixed(5);

Object Prototypes

As in most object oriented programming (OOP) languages the concept of inheritance is key. OOP languages do this via a class/object that inherits from something that is of the same sort. For instance if there was a class called Clothing that contained properties like: color, material, weight. We could use the properties of that class to make even more specific classes that have not only the properties listed in the original class but also unique properties for only objects of that unique type. For instance, consider a shoe class and a shirt class. Both of those items are considered clothing and both have properties like color, material, and weight, but they also have unique properites. Shoes might have a unique property called laceColor and shirts might have a unique property called frontPocket. Using inheritance developers may avoid rewriting the same code and use one source of truth for changes to the software in the future. In JavaScript inheritance is accomplished through Prototypes.

Creating Prototypes

In a JavaScript creating a prototype is very simple, but before a prototype can be created the class it is inheriting from must exist.


The code below will demonstrate the creation of an object and a prototype made from that object.

var Shoe = function(shoeSize, laceColor)
{

var Shirt = function(shirtSize, frontPocket)
